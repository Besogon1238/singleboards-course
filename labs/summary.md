# Резюме лабораторных работ
## Курс: Одноплатные компьютеры на базе RISC-V (Allwinner D1)

---

## Лабораторная №1 (бывшая Lab 1)
### Освоение базовых навыков работы с одноплатником

**Цель:** Подключение к плате через UART, выполнение базовых операций в консоли.

**Основные темы:**
- Одноплатные компьютеры на базе Allwinner D1 (Lichee RV Dock, MangoPi MQ Pro)
- Архитектура RISC-V vs x86_64
- Подключение через UART-преобразователь (3.3V)
- Работа с утилитой tio
- Базовые команды Linux: lscpu, free, df, ip, mkdir, cat

**Оборудование:** Lichee RV Dock, UART-преобразователь

---

## Лабораторная №2 (бывшая Lab 2)
### Работа с платой по SSH

**Цель:** Удалённое взаимодействие с одноплатником через SSH.

**Основные темы:**
- Протокол SSH (Secure Shell)
- Протокол SCP для передачи файлов
- Настройка Wi-Fi/Ethernet на плате
- Создание пользователей в Linux
- Конфигурация sshd

**Результат:** Удалённый доступ к плате без UART

---

## Лабораторная №3 (бывшая Lab 3)
### Использование кросс-компилятора

**Цель:** Сборка программ для RISC-V на x86_64 машине.

**Основные темы:**
- Кросс-компиляция и тулчейны
- Установка gcc-riscv64-linux-gnu
- Компиляция: `riscv64-linux-gnu-gcc -o file file.c`
- QEMU user-static (binfmt) для запуска RISC-V бинарей
- Статическая линковка с флагом -static

**Результат:** Исполняемый файл для RISC-V, собранный на x86_64

---

## Лабораторная №4 (бывшая Lab 8)
### Удалённая разработка и изолированная сборка

**Цель:** Настройка удобной среды разработки для RISC-V.

**Основные темы:**
- VS Code с Remote SSH для удалённой разработки
- VSCodium для RISC-V
- Hasher — изолированное окружение для сборки
- QEMU_LD_PREFIX для эмуляции RISC-V
- Настройка VS Code для кросс-компиляции (sysroot, include paths)

**Результат:** Комфортная среда разработки с изолированной сборкой

---

## Лабораторная №5 (бывшая Lab 4)
### Сборка ядра Linux

**Цель:** Изучить структуру ядра и собрать его под RISC-V.

**Основные темы:**
- Ядро Linux: управление ресурсами, файловые системы, сеть
- make menuconfig — интерактивная настройка
- ARCH=riscv, CROSS_COMPILE=riscv64-linux-gnu-
- Патч PREEMPT_RT для реального времени
- Поддержка Wi-Fi модуля RTW88_8723DS
- Сохранение .config и .dtb файлов

**Результат:** Собственное ядро для Lichee RV Dock

---

## Лабораторная №6 (бывшая Lab 5)
### Создание компонентов образа

**Цель:** Подготовить все компоненты для образа ОС.

**Основные темы:**
- Device Tree (DTS/DTB) — описание железа
- Загрузчик U-Boot (SPL + U-Boot Proper)
- OpenSBI — Supervisor Binary Interface
- Сборка u-boot-sunxi-with-spl.bin
- extlinux.conf — конфигурация загрузки
- Rootfs — корневая файловая система (ALT Linux)

**Результат:** Все компоненты образа готовы

---

## Лабораторная №7 (бывшая Lab 6)
### Подготовка носителя и запись образа

**Цель:** Собрать полноценный загрузочный образ и записать на SD-карту.

**Основные темы:**
- Разбивка SD-карты: BOOT (FAT32) + ROOTFS (ext4)
- Запись загрузчика: `dd if=u-boot-sunxi-with-spl.bin of=/dev/sdX bs=1k seek=8`
- mkfs.vfat, mkfs.ext4
- Структура BOOT: Image, .dtb, extlinux/
- Распаковка rootfs: `tar -xJpf rootfs.tar.xz`

**Результат:** Рабочая SD-карта с ALT Linux

---

## Лабораторная №8 (бывшая Lab 7)
### Добавление поддержки SPI-интерфейса

**Цель:** Включить SPI на плате для взаимодействия с периферией.

**Основные темы:**
- SPI (Serial Peripheral Interface): MOSI, MISO, SCK, CS
- Модификация Device Tree (spi1 на выводах PD10-PD15)
- Драйверы ядра: SPI_SUN6I, DMA_SUN6I, SPIDEV
- Активация в .dts файле: `status = "okay"`
- Тестирование замыканием MOSI/MISO

**Результат:** Устройство /dev/spidev* для работы с SPI из userspace

---

## Лабораторная №9 (бывшая Lab 9)
### Добавление поддержки I2C-интерфейса

**Цель:** Включить I2C (TWI) для подключения датчиков и дисплеев.

**Основные темы:**
- I2C/TWI: SDA (данные), SCL (синхронизация), ведущий/ведомый
- Device Tree: i2c2 на пинах из банков PE/PG
- Драйверы ядра: I2C_MV64XXX, I2C_CHARDEV
- Подключение OLED-экрана SSD-1306
- Python-библиотека luma-oled

**Результат:** Устройство /dev/i2c-0, вывод на экран

---

## Лабораторная №10 (бывшая Lab 10)
### Работа с логическими анализаторами

**Цель:** Анализ цифровых сигналов и протоколов.

**Основные темы:**
- Логический анализатор vs осциллограф
- Sigrok/PulseView — программное обеспечение
- Декодирование UART (115200 8N1)
- Библиотека libgpiod2 для GPIO
- GPIO: gpiochip, линии, потребители
- Генератор меандра на GPIO (PE16, линия 144)

**Результат:** Анализ сигналов, генератор импульсов
